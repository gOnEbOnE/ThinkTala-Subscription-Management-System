1. Folder Structure

Struktur folder terbaru memisahkan resource publik dan view secara lebih rapi.

/zaframework
├── main.go               # Entry point
├── .env                  # Environment Config
├── app/
│   ├── modules/         # Modules (Controller, Service, Repo)
│   └── routes/           # Routing System
├── core/                 # Framework Engine
│   ├── utils/            # Helpers (Session, Image, etc)
│   └── http/             # Response & Middleware
└── public/
    ├── assets/           # CSS, JS, Uploaded Files
    └── views/            # HTML Templates

2. Configuration

Pengaturan aplikasi terpusat di file .env. Core engine akan memuat konfigurasi ini saat startup.

# .env Example
app_name=ZAFramework
port=9002
app_env=development

# Database
postgres=true
read_db_host=localhost
read_db_port=5432
read_db_user=postgres
read_db_pass=rahasia

# Session & Redis
SESSION_DRIVER=redis
redis=true
redis_host=localhost

3. Session Manager

Mendukung Multi-Driver: cookie, redis, stateless (JWT), dan stateless_with_redis.

Usage in Controller

import "github.com/master-abror/zaframework/core/utils/session"

func (c *Controller) Login(w http.ResponseWriter, r *http.Request) {
    // Set Session (Otomatis menyesuaikan driver di .env)
    session.Set(w, r, "user_id", 101)
    
    // Get Session
    uid := session.Get(r, "user_id")

    // Logout
    session.Destroy(w, r)
}

4. File Uploader

Menangani upload file dengan validasi tipe, ukuran, dan auto-rename (UUID).

func UploadAvatar(w http.ResponseWriter, r *http.Request) {
    folder := "public/assets/uploads/avatars"
    
    // Params: Request, FieldName, Folder, AllowedExts, MaxSize
    filename, err := utils.UploadFile(r, "foto", folder, utils.FileImages, 2*1024*1024)
    
    if err != nil {
        http.Error(w, err.Error(), 400)
    }
}

5. Image Processing

Resize, Compress, dan Crop gambar secara otomatis menggunakan preset cerdas.

// Setelah upload, proses gambarnya
fullPath := "public/assets/uploads/avatars/" + filename

// Resize jadi kotak 200x200 & Compress Quality 80
err := utils.ProcessImage(fullPath, utils.ImgSquareSmall)

// Atau Custom
custom := utils.ImagePreset{Width: 800, Height: 400, Quality: 70, Mode: "fit"}
utils.ProcessImage(fullPath, custom)

8. Routing System

Sistem routing terpusat di folder app/routes. Menggunakan standard library http.ServeMux yang dibungkus.

Register Route

// File: app/routes/router.go
func Init(app *core.App, loginCtrl *login.Controller) {
    
    // 1. GET Request
    app.Router.HandleFunc("GET /dashboard", loginCtrl.Dashboard)

    // 2. POST Request
    app.Router.HandleFunc("POST /api/login", loginCtrl.LoginAuth)
    
    // 3. Static Files
    fs := http.FileServer(http.Dir("./public/assets"))
    app.Router.Handle("GET /assets/", http.StripPrefix("/assets/", fs))
}

9. Worker Pool (Concurrency)

Fitur andalan ZAFramework untuk memproses tugas berat di latar belakang tanpa memblokir request user.

1. Register Job Handler (main.go)

// Daftarkan tugas "send_email" ke dispatcher
app.RegisterJob("send_email", func(payload interface{}) error {
    email := payload.(string)
    fmt.Println("Sending email to:", email)
    time.Sleep(2 * time.Second) // Simulasi berat
    return nil
})

2. Dispatch Job (Controller)

func (c *Controller) Register(w http.ResponseWriter, r *http.Request) {
    // Proses DB selesai...
    
    // Kirim email di background worker
    // User langsung dapat response tanpa menunggu email terkirim
    c.Dispatcher.Dispatch(concurrency.Job{
        Type:    "send_email",
        Payload: "user@example.com",
    })
    
    c.Response.JSON(w, r, map[string]any{"msg": "Success"})
}

10. Database Bypass Mode

Fitur unik untuk menjalankan aplikasi tanpa koneksi database (Nil-Safe).

Set postgres=false di .env agar aplikasi tetap jalan meskipun DB mati.
