### Service Breakdown

| Service | Port | Fungsi | Routes |
|---------|------|--------|--------|
| **Gateway** | 2000 | Reverse proxy, auth middleware, serve frontend | `/`, `/account/*`, `/ops/*`, `/client/*`, `/compliance/*` |
| **Users** | 2006 | Authentication, Login, Register, OTP | `/api/auth/*`, `/api/verify-otp/*` |
| **Account** | 2001 | User dashboard, profile management | `/api/user/*`, `/api/profile/*` |
| **Notification** | 5003 | Email, SMS, Push notifications | `/api/notify/*` |
| **Subscription** | 5004 | Billing, subscription plans | `/api/subscription/*` |
| **Operational** | 5005 | Operational management | `/api/ops/*` |
| **Tickets** | 2004 | Support ticketing | `/api/tickets/*` |

## рҹ”җ Environment Configuration

### Mandatory Environment Variables

| Service | Variable | Default | Description |
|---------|----------|---------|-------------|
| **All** | `app_env` | development | development / production |
| **All** | `port` | (varies) | Service port |
| **All** | `timezone` | UTC | Asia/Jakarta recommended |
| **All** | `postgres` | true | Enable/disable database |
| **All** | `redis` | true | Enable/disable Redis |
| **Auth** | `SESSION_DRIVER` | cookie | cookie / redis / stateless |
| **Auth** | `JWT_EXPIRED` | 9600 | JWT expiration in seconds |
| **DB** | `read_db_host` | localhost | Database host |
| **DB** | `read_db_port` | 5432 | Database port |
| **DB** | `read_db_user` | postgres | Database user |
| **DB** | `read_db_pass` | (required) | Database password |
| **DB** | `read_db_name` | (required) | Database name |

### Session Drivers Explained

```env
# 1. COOKIE (Default - Recommended untuk simple setup)
SESSION_DRIVER=cookie
# Pro: Stateless, no server storage needed
# Con: Limited to ~4KB per user

# 2. REDIS (Recommended untuk production)
SESSION_DRIVER=redis
# Pro: Scalable, fast, distributed sessions
# Con: Requires Redis running

# 3. STATELESS (Pure JWT)
SESSION_DRIVER=stateless
# Pro: Maximum scalability, no session storage
# Con: Cannot invalidate tokens

# 4. STATELESS_WITH_REDIS (Hybrid)
SESSION_DRIVER=stateless_with_redis
# Pro: JWT token + session data in Redis
# Con: Complexity
```

```go
// users/app/modules/login/controller.go
func (c *Controller) Auth(w http.ResponseWriter, r *http.Request) {
    // 1. Extract email & password from request
    email := r.FormValue("email")
    password := r.FormValue("password")

    // 2. Query database
    user, err := c.userRepo.FindByEmail(email)
    if err != nil {
        return error(401, "Invalid credentials")
    }

    // 3. Verify password (bcrypt)
    if !utils.VerifyPassword(user.PasswordHash, password) {
        return error(401, "Invalid credentials")
    }

    // 4. Generate JWT token
    jwtToken, err := utils.GenerateJWT(user.ID, user.Email, user.Role)
    if err != nil {
        return error(500, "Token generation failed")
    }

    // 5. Store JWT in Redis with encrypted key
    tokenKey := utils.GenerateUUID()  // e.g., "Z6fUjApoSS0T4a7RPU2Y"
    redisKey := fmt.Sprintf("session:%s", tokenKey)
    redis.Set(redisKey, jwtToken, 7*24*time.Hour) // 7 days

    // 6. Set secure cookie with encrypted token key
    http.SetCookie(w, &http.Cookie{
        Name:     "token",
        Value:    tokenKey,  // Encrypted in middleware
        Path:     "/",
        MaxAge:   604800,
        HttpOnly: true,
        Secure:   true,
        SameSite: http.SameSiteLax,
    })

    // 7. Redirect to dashboard
    w.Header().Set("Location", "/ops/dashboard")
    w.WriteHeader(http.StatusFound)
}
```

```go
// gateway/auth/auth.go
func GetUserFromToken(r *http.Request) (*User, error) {
    // 1. Read cookie
    cookie, err := r.Cookie("token")
    if err != nil {
        return nil, fmt.Errorf("token cookie not found")
    }

    // 2. Decrypt cookie value to get UUID
    uuid, err := DecryptToken(cookie.Value)
    if err != nil {
        return nil, fmt.Errorf("failed to decrypt token: %v", err)
    }

    // 3. Query Redis
    redisKey := fmt.Sprintf("session:%s", uuid)
    jwtToken, err := redis.Get(redisKey)
    if err != nil {
        return nil, fmt.Errorf("redis disabled")  // Token not found
    }

    // 4. Validate JWT signature
    claims, err := utils.ValidateJWT(jwtToken)
    if err != nil {
        return nil, fmt.Errorf("invalid token: %v", err)
    }

    // 5. Extract user info from JWT claims
    return &User{
        ID:       claims.UserID,
        Email:    claims.Email,
        RoleCode: claims.Role,
    }, nil
}
```

## рҹӣ пёҸ Troubleshooting

### Problem: Gateway Crashes on Startup

**Error:** `panic: runtime error: invalid memory address or nil pointer dereference`

**Solution:**
1. Add `config, err = loadConfig()` di gateway `main()` sebelum route registration
2. Pastikan `gateway/routes.json` exist dan valid JSON
3. Check: `go run main.go` output untuk error message

### Problem: Login Returns `redis disabled`

**Error:** `[AUTH] Token error: redis disabled`

**Solution:**
```env
# Pastikan di `.env` file:
redis=true
redis_host=localhost
redis_port=6379
redis_pass=          # KOSONG (jika redis tidak pakai password)
redis_db=0
```

### Problem: "ERR Client sent AUTH, but no password is set"

**Cause:** Account service mengirim password ke Redis yang tidak pakai password

**Solution:**
```env
# account/.env
redis_pass=          # HARUS KOSONG

# users/.env
redis_pass=          # HARUS KOSONG
```

### Problem: JWT Key Not Found

**Error:** `[FATAL] JWT keys not found: open certs/private.pem`

**Solution:**
```powershell
cd C:\PROPENSUY\thinknalyze
go run generate_certs.go
```

### Problem: Infinite Redirect Loop

**Symptoms:**
```
GET /ops/dashboard вҶ’ 302
GET /account/login вҶ’ 303
GET /account/wrapper вҶ’ 302
... (repeat)
```

**Cause:**
- Accessing service directly (e.g., `localhost:2006`) instead of through gateway
- Token not properly stored in Redis
- Session cookie name mismatch

**Solution:**
1. Always access via `http://localhost:2000` (gateway)
2. Never access `localhost:2006` directly in browser
3. Check Redis has token: `redis-cli get session:<uuid>`
4. Verify `SESSION_NAME` matches in all services

### Problem: CORS Errors

**Error:** `Access to XMLHttpRequest blocked by CORS policy`

**Solution:**
```json
// gateway/routes.json
{
  "allowedOrigins": ["*"],  // For development
  "routes": [
    {
      "path": "/api/auth",
      "cors": true  // Enable CORS for this route
    }
  ]
}
```

### Problem: Database Connection Failed

**Error:** `dial tcp localhost:5433: connectex: No connection could be made`

**Solution:**
```powershell
# 1. Check PostgreSQL is running
psql -U office -d office_db -c "SELECT 1"

# 2. Verify connection details in .env
postgres=true
read_db_host=localhost
read_db_port=5433
read_db_user=office
read_db_pass=M3l4t1cn
read_db_name=office_db

# 3. Check PostgreSQL listening port
psql -U postgres -c "SHOW port"
```
